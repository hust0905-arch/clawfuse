## 1. 项目概述

ClawFUSE 将华为 Drive Kit 云存储挂载为本地 FUSE 文件系统，为 OpenClaw 容器内的 AI Agent 提供透明的文件读写能力。核心目标：**容器秒级启动、Agent 立即可用、数据按需加载、写入持久化。**

替代原始方案（容器启动时下载完整压缩包再解压），解决 20GB+ 工作空间需要 3-10 分钟启动的问题。

## 4. 性能成果

### 4.1 核心指标

| 指标 | ClawFUSE | 原始方案 (20GB) | 加速比 |
|------|----------|----------------|--------|
| 容器启动到可用 | **260ms** | 227s | **873x** |
| 首次访问深文件 | **342ms** | 227s | **663x** |
| BFS 全量加载 1111 目录 | **19s** | 227s | **12x** |
| 缓存命中 stat | **0.002ms** | 0.1ms | **50x 快于本地** |
| 缓存命中读 5MB | **0.4ms** | 5ms | **12x 快于本地** |
| 并发 readdir | **2629 QPS** | ~1000 | **2.6x** |

### 4.2 冷启动元数据加载（v4 基准测试）

| 目录数 | BFS 加载 | 首次 stat | 首次 read | BFS后 walk |
|--------|---------|-----------|-----------|-----------|
| 85 | **1.0s** | 154ms | 390ms | 0.0s |
| 259 | **4.0s** | 324ms | 352ms | 0.1s |
| 1111 | **19.0s** | 342ms | 427ms | 0.4s |

BFS 加载时间与目录数线性相关，平均 ~15ms/目录，可按此估算任意规模。

### 4.3 关键结论

1. **启动时间与数据规模无关** — 1GB 或 100GB 工作空间，挂载都是 260ms
2. **首次访问亚秒级** — 3 层深文件 342ms 即可达
3. **缓存命中后优于本地磁盘** — stat 快 50x，5MB 读取快 12x
4. **写入不阻塞** — 内存缓冲 < 1ms，异步上传不感知网络延迟

## 5. 部署验证

### 5.1 已验证环境

| 环境 | 角色 | 结果 |
|------|------|------|
| 腾讯云 4C8G Ubuntu (root) | 测试服务器 | **全部通过** |
| K8s 普通容器 (非 root) | 目标环境 | FUSE 权限待解决 |
| K8s 安全容器 (Kuasar) | 目标环境 | FUSE 不支持 |

### 5.2 部署阻塞问题

| 问题 | 状态 | 解决方案 |
|------|------|----------|
| 普通容器 `/dev/fuse` 权限不足 | **待解决** | K8s YAML 加 SYS_ADMIN + /dev/fuse |
| 安全容器无 FUSE 内核模块 | **待解决** | 需平台团队开启，或走 CSI 模式 |
| 安全容器无法 `privileged` | 待安全团队审批 | 最小权限方案：仅 SYS_ADMIN + /dev/fuse |

## 6. 风险与待办

### 6.1 高优先级

| 项目 | 说明 | 计划 |
|------|------|------|
| **容器 FUSE 权限** | 需平台团队配合修改 K8s YAML 或 Pod 配置 | 推进中 |
| **CSI Driver 方案设计** | 安全容器的长期解决方案 | 下一阶段 |
| **Drive Kit 云空间配额** | 大量测试后出现 SPACE_FULL，删除后配额未立即释放 | 需确认回收机制 |

### 6.2 中优先级

|------|------|

| 内网部署验证 | Drive Kit API 就近部署后性能预期提升 10-160x |

| 项目 | 说明 |
